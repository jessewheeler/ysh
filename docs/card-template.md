# Membership Card Template — Seasonal Update Guide

The YSH membership card is generated by stamping each member's name onto the
official Sea Hawkers Booster Club card template. The template itself is a PNG
committed to the repo; everything else on the card (logo, chapter name, board
member signature, expiry date) is baked into that image.

## Files

| File                                     | Purpose                                           |
|------------------------------------------|---------------------------------------------------|
| `public/img/card-template.png`           | Current season's card background (committed)      |
| `data/cards/YSH-Card<YY>-<YY>.pdf`       | Source PDF received from national (not committed) |
| `services/card.js`                       | Generates per-member PNGs and PDFs                |
| `db/repos/cards.js`                      | Upserts card rows by `(member_id, year)`          |
| `.claude/skills/update-card-template.md` | Claude runbook for seasonal template updates      |

## Every new season: replace the template

1. **Obtain the new card PDF** from the Sea Hawkers national club
   (usually distributed to chapter presidents before the season kicks off).

2. **Place the PDF** in `data/cards/` and note its filename, e.g.
   `YSH-Card26-27.pdf`.

3. **Convert to PNG** and crop to just the card area:

   ```bash
   # Convert at 300 DPI
   gs -dNOPAUSE -dBATCH -sDEVICE=png16m -r300 \
     -sOutputFile=data/cards/raw-300dpi.png \
     data/cards/YSH-Card26-27.pdf

   # Auto-crop whitespace and add a 20 px margin
   magick data/cards/raw-300dpi.png -trim -bordercolor white -border 20 \
     public/img/card-template.png

   # Confirm dimensions
   identify public/img/card-template.png
   ```

4. **Check the result** — open `public/img/card-template.png` and verify it
   looks correct.

5. **Calibrate the name field** (see section below).

6. **Commit** the new template PNG:

   ```bash
   git add public/img/card-template.png
   git commit -m "chore: update membership card template for 26-27 season"
   ```

7. **Clean up** temporary files:

   ```bash
   rm data/cards/raw-300dpi.png
   ```

## Calibrating the name field

After replacing the template PNG you need to verify (and possibly adjust) the
pixel coordinates in `services/card.js` where the member name is stamped.

The relevant constants are near the top of the file:

```js
const NAME_X = 300;         // left edge of name text (after "Member Name:" label)
const NAME_Y = 344;         // canvas baseline — sits on the underline
const NAME_MAX_WIDTH = 680; // clipping width
const NAME_FONT_SIZE = 36;  // px on the 1008×557 canvas
```

### Quick calibration workflow

1. Generate a test card for any member through the admin UI
   (`/admin/members/<id>` → "Regenerate Card"), then download it.

2. Open the PNG and check whether the name lands on the underline.

3. Adjust `NAME_X` / `NAME_Y` in `card.js` and regenerate until it looks right.

    - Increase `NAME_X` to move the text right.
    - Increase `NAME_Y` to move the text down (canvas baseline).
    - Increase `NAME_FONT_SIZE` if the text looks too small.

4. Re-run the tests to confirm nothing is broken:

   ```bash
   npx jest --forceExit --testPathPatterns=card
   ```

### ImageMagick annotation method

Draw horizontal reference lines on the template to visually locate the underline:

```bash
magick public/img/card-template.png +repage \
  -fill none -stroke red -strokewidth 1 \
  -draw "line 0,340 1008,340" \
  -draw "line 0,350 1008,350" \
  -draw "line 0,360 1008,360" \
  /tmp/calibration.png
open /tmp/calibration.png
```

Use `+repage` to strip virtual canvas offsets from Ghostscript before measuring.

## Card downloads

Admin card downloads (`/admin/members/:id/card/pdf` and `/card/png`) proxy
remote B2 files through the server with `Content-Disposition: attachment`.
Download filenames follow the pattern `YSH-FirstName-LastName-Year.ext`.

## Notes

- The template PNG dimensions **must match** `CARD_WIDTH` / `CARD_HEIGHT` in
  `card.js` (currently `1008 × 557`). If the new season's card crops to a
  different size, update those constants and re-calibrate.

- The PDF output is rendered at `PDF_SCALE = 0.5` (504 × 279 pt). Name
  coordinates are scaled automatically — you only need to calibrate the
  PNG-canvas values.

- `data/` is gitignored; never commit the raw PDF or intermediate PNGs from
  there.

- The card upsert in `db/repos/cards.js` looks up existing rows by
  `(member_id, year)`. Because `NULL = NULL` is false in SQL, the lookup
  handles NULL years with a separate `year IS NULL` query branch. Ensure
  `membership_year` is set on the member before generating cards to avoid
  null-year filenames like `card-1-null.png`.
